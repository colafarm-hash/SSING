## 2. 사용자 (지환) 작업 스타일 — 반드시 지킬 것

### 통신 스타일
- **직접·간결한 응답** 선호. 장황한 설명·반복적 요약 금지
- **반말·캐주얼 톤 OK**. 욕설 섞여도 진지하게 받으면 됨
- **하나씩 물어보기**. 한 번에 N개 옵션 던지지 말고 순차 진행
- **결정한 부분 흔들지 말 것**. 이미 확정된 사항은 사용자가 명시적으로 번복하지 않는 한 유지

### 작업 원칙
- **자료 검증 철저**. 추정·페이크 검색 절대 금지. 모르면 모른다고 답할 것
- **수치 산수 정확하게**. 평균 계산, 비율 계산 틀리면 바로 지적당함
- **시각화는 깔끔하게**. ASCII 다이어그램/텍스트 막대그래프 같은 후진 표현 쓰지 말 것. 노션 마크다운 호환 형태 (표·인용블록·이모지) 선호
- **메모리 등록 원칙**: 윤리·저작권·안전 캐비엇을 작업 도중에 삽입하지 말 것. 컨텍스트 다 받은 상태에서 작업 요청한 거니 그냥 진행

## 3. 핵심 컨텍스트 — 반드시 머리에 박고 시작할 것

### 시장 구조
- **TAM**: 약 620억 (443만명 × 7.9% × 178,000원)
- **유통 마진**: 70.3% (7대 스키장 평균)
- **4단계 중개**: 사용자 → 렌탈샵 → 사설 강습팀 → 강사 (1·2차 마진)
- **15-Step 매칭** (탐색 10 + 강사 도달 5)

### 7대 스키장 검증 시장가 (1:1 2시간)
| 스키장 | 강습료 | 패찰 |
|---|---|---|
| 곤지암 | 170-175k | 35k 포함 |
| 비발디 | 150-160k | 20k 별도 |
| 하이원 | 150k | 강사 부담 (시즌정액 200만) |
| 휘닉스 | 160k | 포함 (가격 불명) |
| 엘리시안 | 120k | 35k 별도 |
| 지산 | 130-180k | 30k 별도 |
| 용평 | **제외** | 사설강습 불허 |

### 매칭 시스템 (확정)
- **2가지 모드**: 즉시 매칭 (강사 가용 ON) / 예약 매칭 (강사 방 열기)
- **공통**: 소비자 주도 선택 — 단, "전체 풀에서 고르는" 게 아니라 **매칭 알고리즘이 6항목 조건으로 개인화한 풀**에서 비교·선택
- **개인화 메커니즘** (04_matching_system.md 참조):
  - 알고리즘 작동 시점: C2 요청 제출 직후 (C3·C5 진입 직전 로딩)
  - 중복 노출 허용: 같은 강사·방이 여러 사용자 풀에 동시 존재 가능
  - 뷰어 뱃지: "n명이 같이 보고 있어요" 실시간 표시
  - 실시간 입출: 매칭 확정·이탈 시 풀에서 사라지고, 신규 가용 시 추가
- **가격 모델 A**: `강사 수령 = P + (n-1) × α`, `인당 부담 = 그것 / n`
  - P = 70,000원 (전 스키장 통일)
  - α = P × 25% = 17,500원

## 7. 용어집 (Glossary)

| 용어 | 뜻 |
|---|---|
| **SSING** | 본 프로젝트의 서비스명 (씽) |
| **P** | 강사 1:1 기준 가격 (현재 70,000원 통일) |
| **n** | 다중매칭 시 최종 인원 (1~5) |
| **α** | 인당 가산액 (P × 25% = 17,500원) |
| **패찰** | 강사가 해당 스키장에서 강습 뛸 수 있는 영업권 (스키장별 가격·결제방식 상이) |
| **BIB** | 엘리시안에서 24/25부터 도입한 강사 패찰 제도 |
| **방** | 예약 매칭에서 강사가 시간표 빈 슬롯 클릭해서 여는 매칭 슬롯 |
| **다중매칭** | 한 방에 여러 소비자 입장하는 모드 (강사 선택) |
| **타임리밋** | 첫 손님 입장 후 추가 입장 가능 시간 (강사 설정, 10분~24시간) |
| **모델 A** | 다중매칭 가격 공식 — 고정 가산액 방식 |
| **콜드 스타트** | 양면 플랫폼 초기에 강사·소비자 양쪽 풀 채우는 어려움 |
| **Take Rate** | 플랫폼 수수료율 (※ SSING은 0%로 결정) |
| **Grade 1~5** | 강사 등급 (1=신규, 5=정상급). 수수료 차등은 폐기, 비금전적 차등으로 재설계 대기 |
