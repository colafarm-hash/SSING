# 매칭 시스템 확정 사항

## 기본 구조

SSING은 **두 가지 매칭 모드**를 운영하며, 각 모드 안에서 **타인 매칭(그룹) 옵션**으로 다시 갈린다.

| 모드 | 트리거 | 시간 범위 | 강사 행위 | 타인 매칭 옵션 |
|---|---|---|---|---|
| 즉시 매칭 | 강사 "지금 가용 ON" | 매칭 후 1시간 내 만남 | 단순 토글 ON/OFF | 사용자가 C2에서 선택 — ON 시 그룹 매칭 + voting / OFF 시 강사 풀 직접 선택 |
| 예약 매칭 | 강사 "방 열기" | 최소 2시간 ~ 최대 3개월 전 | 시간표 빈 슬롯 클릭 → 방 오픈 | 사용자가 C2에서 선택 — "함께도 OK" / "1:1 단독만" |

**공통 원칙**: 둘 다 소비자 주도 선택. 단, "전체 강사 풀에서 고르는" 게 아니라 **매칭 알고리즘이 사용자 조건에 맞춰 개인화한 풀**에서 비교·선택.

## 매칭 알고리즘 (개인화 매칭)

### 핵심 원칙

SSING의 매칭은 **개인화된 큐레이션**이다. 사용자가 C2에서 요청을 제출하는 순간 매칭 알고리즘이 작동하여, **사용자의 6항목 조건에 부합하는 강사·방만** 선별해 풀에 노출한다. 사용자는 "전체에서 고르는" 느낌이 아니라 "내 조건에 맞춰진 풀"을 받는다.

### 알고리즘 작동 시점

| 모드 | 입력 | 알고리즘 동작 | 출력 |
|---|---|---|---|
| 즉시 매칭 | C2 요청 제출 | 현재 "즉시 가용 ON" 강사 풀 × 사용자 조건 필터링 + 정렬 | C3 강사 풀 (개인화) |
| 예약 매칭 | C2 요청 제출 | 현재 열린 방 × 사용자 조건 필터링 + 정렬 | C5 방 목록 (개인화) |

### 필터링 조건 (요청 조건 기반)

| 조건 | 매칭 기준 |
|---|---|
| 종목 | 정확 일치 (스키 ↔ 보드) |
| 레벨 | 강사·방의 가능 레벨 집합에 사용자 레벨 포함 |
| 인원 | 강사·방의 최대 인원 ≥ 사용자 요청 인원 |
| 시간 길이 | 강사·방이 해당 시간 길이(2/3/4h) 수용 가능 |
| 장소 | 사용자 위치 = 강사·방 등록 스키장 |
| 소비자 정보 | 강사 선호 연령대·성별 필터 (강사가 설정한 경우만) |
| 시작 시간 (예약 모드) | 사용자 요청 시작 시간이 방의 시작 시간과 일치 또는 허용 윈도우 내 |
| 다중매칭 허용 (즉시·예약) | 즉시 + 함께 OK → 강사단위 합류 가능 카드 노출 (강사 방 자동 트리거 또는 합류). 즉시 + 1:1 단독만 → 1:1 단독 흐름. 예약 + 함께 OK → 다중매칭 ON 방 + 1:1 단독 방 모두 노출. 예약 + 1:1 단독만 → 1:1 단독 방만 노출 |

### 결과 정렬 (Phase 1 — 기본)

| 우선순위 | 기준 |
|---|---|
| 1 | 등급 (Grade 높을수록 상단) ※ 등급제 재설계 후 보강 |
| 2 | 평점 |
| 3 | 재예약률 |
| 4 | 거리 (즉시 매칭만) |

> Phase 2 이후 추천 알고리즘(사용자 선호 학습) 도입 가능.

### 풀 노출 메커니즘

#### 중복 노출 허용

- 같은 강사·방을 **여러 사용자에게 동시에** 노출 가능
- 각 사용자는 자기만의 풀을 보지만, 동일 강사·방이 다른 사용자 풀에도 포함될 수 있음
- 강사·방은 매칭이 확정되기 전까지 N개의 사용자 풀에 동시 존재 가능

#### 뷰어 뱃지 — "n명이 같이 보고 있어요"

- 같은 강사·방을 동시에 보고 있는 다른 사용자가 2명 이상일 때 뱃지 노출
- 카피: "**n명이 같이 보고 있어요**" (해요체, 압박감 최소화)
- 컬러: warning 톤 `#FF8A00` 또는 info 톤 `#3478F6` (와프 단계에서 확정)
- 의도: 적정한 희소성 신호로 의사결정 가속 / 부작용 방지 위해 카피 톤 유연하게
- 갱신: 뷰어 수는 실시간 (WebSocket/SSE)

#### 실시간 입출 (Pool Live Update)

**이탈 (풀에서 사라짐)**

| 트리거 | 처리 |
|---|---|
| 강사가 다른 사용자에게 매칭 확정 | 모든 사용자 풀에서 즉시 제거 |
| 강사 "즉시 가용 OFF" 토글 | 즉시 매칭 풀에서 제거 |
| 방 정원 마감 | 예약 풀에서 제거 |
| 강사 방 닫기 | 예약 풀에서 제거 |
| 사용자 조건 외 변화 (예: 강사가 인원 한도 변경) | 해당 사용자 풀에서 재평가 |

**추가 (풀에 들어옴)**

| 트리거 | 처리 |
|---|---|
| 새 강사 "즉시 가용 ON" | 조건 부합 시 사용자 풀에 추가 |
| 새 방 열림 | 조건 부합 시 사용자 풀에 추가 |

**모션**: 이탈/추가 시 카드 fade-out / fade-in 트랜지션. 풀이 살아 움직이는 인상.

#### 락-인 (결제 단계)

- 사용자가 강사·방 선택 → 결제 진행 단계 진입 시 **해당 강사·방은 일시 락**
- 락 상태에서는 다른 사용자가 해당 카드 탭 시 "방금 매칭이 시작됐어요" 안내 (또는 카드 자체가 풀에서 사라짐)
- 결제 실패·이탈 시 락 해제 → 풀 복귀
- 락 유지 시간·정확한 처리는 미확정 (`05_pending_decisions.md` Q-1)

## 즉시 매칭 + 타인 매칭 (강사단위 합류)

> **2026-05-29 메커니즘 변경**: 이전의 "그룹 매칭 → Voting" 모델 폐기. 사용자가 강사를 직접 선택하고, 같은 강사를 선택한 다른 사용자들이 자동으로 합류하는 **강사단위 합류** 모델로 전환.

### 핵심 원칙

**강사단위 매칭**. 사용자가 C3 강사 풀에서 강사를 직접 선택하면, 시스템이 자동으로 그 강사의 "방"을 열거나 기존 방에 합류시킨다. 예약 매칭의 "방" 모델을 즉시 매칭에 **첫 손님 = 자동 트리거** 형태로 적용한 것. 본질적으로 즉시·예약 두 모드는 같은 "강사+시간 슬롯" 메커니즘으로 수렴.

### 작동 흐름

사용자가 C2에서 즉시 모드 + 함께도 OK 선택 시:

```
C2 제출
  ↓
[1] C3 강사 풀 (개인화) — 각 강사 카드에 현재 합류 상태 표시
     "1:1 단독 가능" / "N명 함께 듣는 중 · OO분 남음" / "마감"
  ↓
[2] 사용자가 강사 선택 → C4 강사 프로필
  ↓
[3] "매칭 시작" 탭 → 시스템 분기:
     - 이 강사에 열린 방 없음 → 새 방 자동 생성 (사용자 = 첫 손님). 1시간 윈도우 시작
     - 이미 열린 방 있음 → 합류 동의 다이얼로그 (현재 인원·임시 가격·남은 윈도우)
  ↓
[4] 합류 확정 → C7 결제 → C8 매칭 완료
  ↓
[5] 타임리밋 만료 또는 정원 도달 → 인원 확정 → 가격 차액 정산 (이전 사용자에게 환불 또는 적립)
```

### 강사 방 자동 생성 규칙 (즉시 매칭)

| 항목 | 값 |
|---|---|
| 트리거 | 첫 손님이 C4에서 "매칭 시작" 탭 |
| 윈도우 | 1시간 (즉시 매칭의 강사 가용 윈도우와 동일) |
| 정원 | 5명 (강사 방 최대 인원) |
| 다중매칭 | 자동 ON (사용자가 함께 OK로 진입했으므로) |
| 닫힘 조건 | 윈도우 만료 / 정원 도달 / 강사가 즉시 가용 OFF / 첫 손님이 1:1 단독 선택 |

### 합류 동의 메커니즘 (C4 매칭 시작 직후)

예약 매칭의 C6 동의 다이얼로그와 동일 패턴. 시점별 분기:

| 시점 | 다이얼로그 본문 | 옵션 |
|---|---|---|
| 첫 손님 (현재 인원 = 0) | "다른 사람들과 함께 들으면 가격이 분담돼요" | "함께 들을게요" (방 열기, 후속 합류 가능) / "1:1 단독만" (방 즉시 닫힘) |
| 후속 손님 (현재 인원 ≥ 1) | "OO 강사 방에 현재 N명이 함께 듣고 있어요. 합류하시겠어요?" | "합류할게요" / "다른 강사 보기" (C3 복귀) |

- `allowMulti=false`로 진입한 경우 → 강제로 1:1 단독, 다이얼로그 생략
- 첫 손님이 "1:1 단독만" 선택 시 → 방이 즉시 1:1로 닫힘 (후속 손님 합류 불가)

### C3 강사 풀 — 합류 상태 표시

각 강사 카드에 현재 합류 상태 indicator 추가:

| 상태 | 표시 | 풀 내 동작 |
|---|---|---|
| 방 없음 | "1:1 단독 가능" (neutral chip) | 선택 시 첫 손님 흐름 |
| 방 열림 (정원 미달) | "N명 함께 듣는 중 · OO분 남음" (Brand Blue chip) | 선택 시 합류 흐름 |
| 정원 도달 / 윈도우 만료 | dim 처리 또는 풀에서 제거 | 선택 차단 |

실시간 갱신 (WebSocket/SSE) — 04 매칭 알고리즘의 "실시간 입출" 메커니즘과 동일.

### 매칭 후 가격 (실시간 정산)

가격 모델 A 그대로. 합류 인원이 늘어날 때마다 차액 정산:

```
첫 손님 입장 시점:  1명 → 임시 ₩70,000원 부과
2명째 합류:         인당 ₩43,750 → 첫 손님에게 ₩26,250 환불·적립
3명째 합류:         인당 ₩35,000 → 두 명에게 각 ₩8,750 정산
4명째 합류:         인당 ₩30,625 → 세 명에게 각 ₩4,375 정산
5명째 합류 (정원):  인당 ₩28,000 → 네 명에게 각 ₩2,625 정산 → 인원 확정
또는 타임리밋 만료 시점에 현재 인원으로 확정
```

### 즉시 매칭 3가지 경로 정리

| 구분 | 입력 | 흐름 | 강사 방 |
|---|---|---|---|
| 즉시 1:1 단독 | 인원 1 + allowMulti=false | C3 → C4 → C7 | 1:1 즉시 생성 + 닫힘 |
| 즉시 직접 그룹 | 인원 2~5 + allowMulti=false | C3 → C4 → C7 | 본인 그룹만, 후속 합류 불가 |
| 즉시 함께 OK (강사단위 합류) | 인원 1 + allowMulti=true | C3 → C4 → 합류 분기 → C7 | 자동 트리거 / 기존 방 합류 |

### 예약 매칭과의 관계 (메커니즘 통합)

| 모드 | 방 생성 트리거 | 시작 시간 | 다중매칭 |
|---|---|---|---|
| 예약 | 강사가 명시적으로 "방 열기" | 강사가 선택 (현재 +2h ~ +3개월) | 강사 설정 (ON/OFF) |
| 즉시 함께OK | 첫 손님이 강사 선택 (자동) | 자동 (요청 시점부터 1시간) | 자동 ON |

본질: 두 모드 모두 **"강사 + 시간 슬롯" 단위 방**으로 매칭. 차이는 방 열기 주체만 (강사 명시 vs 첫 손님 자동 트리거). 화면 흐름·결제·차액 정산 메커니즘은 동일.

### 폐기된 voting 메커니즘 (참고)

- 이전 설계: 그룹 매칭 → 강사 voting → 최다 득표 선정
- 폐기 사유: (1) 사용자가 강사 결정권 잃음, (2) 타임리밋 압박이 매칭 경험 해침, (3) 동률 처리·기권 등 복잡도 ↑, (4) 강사단위 합류가 예약 모드와 메커니즘 통합되어 더 단순
- 따라서 `GroupMatching.tsx` (Phase A/B/C), `/group-matching` 라우트, voting UI 전부 제거

### 미확정 사항

- 합류 가능 윈도우 정확값 (1시간이 기본이지만 강사 가용 OFF 시점에 따라 변동)
- 결제 진행 중 다른 사용자 합류 시 가격 재산정 타이밍 (결제 완료 후 정산? 즉시?)
- 정원 도달 직전 합류 시점에서 락-인 (다른 사용자 차단 시간)
- 합류 후 사용자 이탈 (결제 실패 / 강제 종료) — 가격 재정산 방향
- C3 풀에서 1:1 단독 강사 카드와 합류 가능 카드의 정렬 (분리? 통합?)
- 합류 윈도우 카운트다운 표시 정확도 (실시간 vs 1분 단위)

## 강습 요청 조건 (공통 6항목 + 다중매칭 옵션 + 예약 모드 1항목)

### 공통 (즉시·예약 모두)

| 항목 | 내용 |
|---|---|
| 종목 | 스키 / 보드 택1 (강사도 매칭별 택1) |
| 레벨 | 입문 / 초급 / 상급 + 자유 텍스트 (선택) |
| 인원 | 1:1 ~ 1:5 (강사 방 열 때 최대 인원 설정) |
| 시간 길이 | 2시간 / 3시간 / 4시간 |
| 장소 | 자동 위치 감지 (슬로프/만남 지점은 매칭 후 채팅) |
| 소비자 정보 | 강습받는 사람의 나이 + 성별 (다수일 때 각각 입력) |
| **다중매칭 허용 여부** | "함께 OK" (다른 사람과 같은 방에 들어가도 좋음, 가격 분담) / "1:1 단독만" (우리만 듣는 방) — 기본값 "함께 OK" |

> **다중매칭 적용**:
> - 즉시 모드 + 함께 OK → 강사단위 합류 메커니즘 작동 (위 섹션 참조)
> - 즉시 모드 + 1:1 단독만 → 1:1 단독 매칭, 후속 합류 불가
> - 예약 모드 + 함께 OK → 다중매칭 ON인 방까지 노출 (C5 필터)
> - 예약 모드 + 1:1 단독만 → 1:1 단독 방만 노출

### 예약 모드 전용

| 항목 | 내용 |
|---|---|
| 시작 시간 | 날짜 + 시간. 가능 범위: 현재 +2시간 ~ +3개월 (방 열기 가능 범위와 동일) |

> 즉시 모드는 시작 시간 = 요청 시점부터 1시간 내로 자동 고정 (사용자 입력 없음).

## 예약 매칭 "방" 모델

### 강사 방 열 때 설정 항목

- 시간 (시간표 빈 슬롯)
- 종목 (스키/보드)
- 가능 레벨 (다중 선택 가능)
- 최대 인원 (1~5)
- 다중매칭 받음/안받음
- 가격 P
- 타임리밋 (다중매칭 ON 시 강사 자율 선택)

### 타임리밋 옵션

| 옵션 | 제약 |
|---|---|
| 10분 / 30분 / 1시간 / 2시간 / 6시간 | 제약 없음 |
| 12시간 / 24시간 | 최소 3일 전 예약일 때만 선택 가능 |

> 선택 가능 = min(강사 선택값, 강습 시작까지 남은 시간)

### 다중매칭 동의 메커니즘

- **첫 손님 입장 시**: "다른 사람을 추가로 받고 페이를 나눠 내시겠습니까?" → 동의/거부
- **2~N번째 손님 입장 시**: "다른 사람들과 함께 들어야 하는 방입니다, 괜찮으십니까?" → 동의/거부
- **레벨 불일치 손님**: 입장 자체 차단 (방 노출 단계에서 필터)
- **첫 손님 거부 시**: 방이 즉시 1:1로 닫힘

### 인원 확정 메커니즘

- 첫 손님 입장 → 타임리밋 카운트 시작
- 정원 도달 OR 타임리밋 만료 → 인원 확정 → 가격 확정
- 결제는 인원 확정 후 일괄 처리
- 방 안에 "확정" 마크 표시

## 가격 모델 — 모델 A (확정)

### 공식

```
강사 총수령액 = P + (n - 1) × α
인당 소비자 부담 = [P + (n - 1) × α] / n
```

| 기호 | 뜻 |
|---|---|
| P | 강사 1:1 기준 가격 (전 스키장 통일: **70,000원**) |
| n | 최종 인원 (1~5) |
| α | 인당 가산액 (= **P × 25% = 17,500원**) |

### 예시 계산 (P=70,000, α=17,500)

| 인원 | 강사 수령 | 인당 부담 | 강사 증가율 | 인당 절감률 |
|---|---|---|---|---|
| 1명 | 70,000원 | 70,000원 | - | - |
| 2명 | 87,500원 | 43,750원 | +25% | -37.5% |
| 3명 | 105,000원 | 35,000원 | +50% | -50% |
| 4명 | 122,500원 | 30,625원 | +75% | -56.3% |
| 5명 | 140,000원 | 28,000원 | +100% | -60% |

> ⚠️ 무료화 결정으로 SSING 수수료는 0%. 강사 수령 = 강사 실수령 (단, 패찰비는 강사 부담)

## 등급제 (재설계 대기)

수수료 차등 (5%~20%) 모델은 무료화로 폐기됨. **비금전적 차등 메커니즘**으로 재설계 필요.

### 재설계 후보 차등 메커니즘

- [ ] 가격 슬라이더 상한 차등 (Grade 5는 최대 P 책정 가능)
- [ ] 풀 노출 우선순위 (높은 등급 강사가 상단 노출)
- [ ] 다중매칭 정원 차등 (Grade 5는 1:5까지, Grade 1은 1:2까지)
- [ ] 배지/타이틀 (브랜드 가치)

### 등급 결정 기준 (유지)

- 유저 만족도 (평점 + 재예약률) × 앱 내 누적 강습량

## 매칭 시스템 통합 흐름

### 양면 가치 흐름

| 흐름 | 소비자 → SSING | SSING → 강사 |
|---|---|---|
| 요청 | 강습 요청 (위치·종목·레벨·시간) | 가용 풀에 자동 노출 |
| 알고리즘 | 6항목 기반 필터 + 정렬 적용 | 조건 부합 시 사용자 풀에 자동 포함 (다중 사용자 풀에 동시 노출 가능) |
| 매칭 | 개인화 풀에서 선택 | 매칭 확정 시 모든 사용자 풀에서 즉시 제외 |
| 결제 | 강습료 결제 | 정산 (패찰비 강사 부담) |
| 사후 | 평가·재예약 | 평가 누적 → 등급 변동 |
